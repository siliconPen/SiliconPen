Ultrasonic Sensor (HC-SR04)

“How Arduino Uses Sound Waves and Timing to Measure Real-World Distance”

So far, Arduino has been reacting to buttons and knobs.
Now we’ll make it measure distance.

The ultrasonic sensor is perfect for this because:

  • it’s visual

  • it’s intuitive

  • it teaches timing, not just voltage

And timing is a big deal in embedded systems.

📡 What Is an Ultrasonic Sensor?

An ultrasonic sensor measures distance using sound.

Not audible sound —
ultrasonic waves (around 40 kHz).

The idea is simple:

  • Send out a sound pulse

  • Wait for it to bounce back

  • Measure how long it took

  • Convert time → distance

This is called time-of-flight measurement.

🧠 HC-SR04 Pin Overview

The HC-SR04 has 4 pins:

  • VCC → 5V

  • Trig → Trigger pin (input from Arduino)

  • Echo → Echo pin (output to Arduino)

  • GND → Ground

Two pins control everything:

  • one starts the measurement

  • one reports the result

🔌 Wiring the Sensor

Connect the sensor like this:


HC-SR04 VCC → Arduino 5V
HC-SR04 GND → Arduino GND
HC-SR04 Trig → Arduino Pin 9
HC-SR04 Echo → Arduino Pin 10

You can use any digital pins, but we’ll stick with these for consistency.

⚙️ How Measurement Actually Works

Here’s what happens internally:

  • Arduino sends a 10 µs HIGH pulse on the Trig pin

  • Sensor emits 8 ultrasonic bursts

  • Sound hits an object and reflects back

  • Echo pin goes HIGH for the duration of the round trip

  • Arduino measures this time

Distance depends on how long Echo stays HIGH.

⏱️ Measuring Time with pulseIn()

Arduino provides a helper function:

pulseIn(pin, HIGH);

It measures how long a pin stays HIGH (in microseconds).

We’ll use this to capture the echo time.

💻 Basic Distance Measurement Code

const int trigPin = 9;
const int echoPin = 10;

void setup() {
  pinMode(trigPin, OUTPUT);
  pinMode(echoPin, INPUT);
  Serial.begin(9600);
}

void loop() {
  digitalWrite(trigPin, LOW);
  delayMicroseconds(2);

  digitalWrite(trigPin, HIGH);
  delayMicroseconds(10);
  digitalWrite(trigPin, LOW);

  long duration = pulseIn(echoPin, HIGH);

  float distance = duration * 0.034 / 2;

  Serial.print("Distance: ");
  Serial.print(distance);
  Serial.println(" cm");

  delay(500);
}

🧮 Where That Formula Comes From

Speed of sound in air ≈ 343 m/s

Convert to cm/µs:

0.034 cm per microsecond

We divide by 2 because:

  • sound travels to the object

  • and back again

🧠 What You Should Observe

  • Distance updates smoothly

  • Close objects → smaller values

  • Far objects → larger values

  • No object → very large or unstable readings

This sensor depends heavily on environment.

⚠️ Common Problems & Fixes

  • Always reading 0 → Echo pin miswired

  • Huge random values → no object in range

  • Unstable readings → angled surfaces or soft materials

  • Wrong distance → sensor too close (<2 cm)

Ultrasonic sensors don’t see everything equally.
